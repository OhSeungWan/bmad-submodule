## Executive Summary

전 직원이 직무에 상관없이 데이터 기반 사고가 가능해지고, 전 퍼널을 이해하며, 실제 성과로 이어지는 데이터 분석 전문 모듈입니다.

**Module Category:** Data Analytics & Insights
**Target Users:** PM, 개발자, 디자이너, 비개발 직군 (마케터, 기획자, 경영진 등)
**Complexity Level:** Medium (단일 에이전트, 다중 워크플로우)

---

## Module Identity

### Module Code & Name

- **Code:** `dae`
- **Name:** `DAE: Data Analysis Expert`

### Core Concept

DAE는 Amplitude MCP를 통해 실제 데이터를 분석하고, 서비스의 전체 퍼널과 실험 이력을 완벽히 이해하는 데이터 분석 전문가 에이전트입니다. 데이터 분석을 모르는 사용자도 차근차근 함께 분석을 진행할 수 있도록 안내합니다.

### Personality Theme

**전문 분석팀** — 데이터 사이언티스트, 분석가 페르소나. 신뢰감 있고 전문적이면서도 따뜻하고 친근한 톤.

## User Scenarios

### Target Users

| 페르소나 | 역할 | 스킬 레벨 | 주요 니즈 |
|----------|------|----------|----------|
| **PM** | 프로덕트 매니저 | 중급 | 데이터 기반 제품 의사결정, 빠른 인사이트 |
| **개발자** | 백엔드/프론트엔드 | 중급 | 기능 변경 영향도 분석, 안전한 배포 |
| **디자이너** | UX/UI 디자이너 | 초급 | 퍼널 UX 개선, 과거 실험 학습 |
| **비개발 직군** | 마케터, 기획자, 경영진 | 초급 | 쉬운 데이터 접근, 자율적 분석 |

### Primary Use Case

1. **비전문가 분석** — 데이터 분석 경험이 없는 팀원도 쉽게 인사이트 획득
2. **영향도 예측** — 기능 변경 전 예상 영향 파악
3. **실험 설계** — A/B 테스트 가설 수립 및 설계
4. **위험 모니터링** — 이상 징후 조기 감지

### User Journey

**시나리오: 디자이너의 퍼널 개선**

```
1. 문제 직면
   └─ 디자이너가 특정 퍼널의 워딩/인터랙션을 바꾸고 싶다
   └─ 하지만 과거 실험 이력, 효과적인 패턴을 모르겠다

        ↓

2. 에이전트에게 분석 요청
   └─ "이 퍼널에서 어떤 실험이 있었어? 뭐가 효과적이야?"

        ↓

3. 에이전트 응답
   ├─ "해당 퍼널에서 [날짜]에 [실험]이 있었고, 결과는 [이랬다]"
   ├─ "인사이트: [요렇고 저렇다]"
   ├─ "제안: [이렇게 하면 더 좋을 것으로 예상]"
   └─ 또는 "이런 실험을 설계하면 더 명확해질 수 있다" 제안

        ↓

4. 실행 & 성과
   └─ 데이터 기반으로 변경/실험 진행
   └─ 실제 성과 상승!
```

## Agent Architecture

### Agent Count Strategy

**단일 에이전트** — DAE의 핵심 가치인 "차근차근 함께 분석"은 한 명의 믿을 수 있는 전문가와 대화하는 경험입니다. 비전문가 사용자에게 여러 에이전트 간 핸드오프는 혼란을 줄 수 있으며, 데이터 분석 도메인은 분석 → 실험 설계 → 모니터링이 하나의 대화에서 자연스럽게 연결됩니다.

### Agent Roster

| Agent | Name | Role | Expertise |
|-------|------|------|-----------|
| data-analyst | Alex | 데이터 분석 전문가 | 데이터 분석, 실험 설계, 인사이트 도출, 위험 모니터링 |

### Agent Interaction Model

단일 에이전트이므로 에이전트 간 상호작용은 없습니다. Alex가 모든 데이터 관련 요청을 처리하며, 다양한 워크플로우를 통해 기능을 분리합니다.

### Agent Communication Style

- **톤:** 친절하고 전문적, 차근차근 설명
- **특징:** 비전문가도 이해할 수 있는 언어 사용
- **메모리:** `hasSidecar: true` — 서비스 맥락, 실험 이력, 사용자 선호 기억

## Workflow Ecosystem

### Core Workflows (Essential)

| 워크플로우 | 목적 | Input → Output |
|-----------|------|----------------|
| **데이터 분석** | Amplitude 데이터 조회 및 분석 | 질문 → 데이터 기반 답변 |
| **실험 설계** | A/B 테스트 설계, 가설 수립 | 목표 → 실험 계획서 |
| **인사이트 도출** | 과거 실험/데이터에서 패턴 발견 | 퍼널/기능 → 인사이트 리포트 |

### Feature Workflows (Specialized)

| 워크플로우 | 목적 | Input → Output |
|-----------|------|----------------|
| **퍼널 분석** | 특정 퍼널 심층 분석 | 퍼널명 → 전환율, 병목, 제안 |
| **실험 결과 분석** | A/B 테스트 결과 해석 | 실험 ID → 결과, 인사이트 |
| **영향도 분석** | 기능 변경 시 예상 영향 | 기능/변경 → 영향 범위, 위험도 |
| **위험 모니터링** | 이상 징후 감지 및 알림 | 설정 → 알림, 리포트 |

### Utility Workflows (Support)

| 워크플로우 | 목적 | Input → Output |
|-----------|------|----------------|
| **서비스 컨텍스트 설정** | 퍼널, 지표 정의 학습 | 서비스 정보 → 저장된 컨텍스트 |
| **리포트 생성** | 분석 결과 문서화 | 분석 결과 → 마크다운/PDF |

### Workflow Connections

```
[서비스 컨텍스트 설정] ← 최초 1회
        ↓
[데이터 분석] ←→ [퍼널 분석]
        ↓
[인사이트 도출] → [실험 설계]
        ↓
[실험 결과 분석] → [영향도 분석]
        ↓
[위험 모니터링] (상시)
        ↓
[리포트 생성] (필요시)
```
