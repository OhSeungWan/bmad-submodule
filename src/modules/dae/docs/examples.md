## Example Workflows

### 예시 1: 디자이너의 퍼널 개선

**상황:** 디자이너가 특정 퍼널의 워딩/인터랙션을 바꾸고 싶지만, 과거 실험 이력과 효과적인 패턴을 모름

**대화:**
```
사용자: 이 퍼널에서 어떤 실험이 있었어? 뭐가 효과적이야?

재만: 해당 퍼널에서 지난 6개월간 3개의 실험이 있었습니다:

1. [2025-11] 버튼 색상 변경 실험
   - 결과: 전환율 +2.3% (유의미)
   - 인사이트: 대비가 높은 색상이 효과적

2. [2025-12] CTA 문구 변경 실험
   - 결과: 전환율 -0.5% (유의미하지 않음)
   - 인사이트: 문구보다 위치가 더 중요

3. [2026-01] 입력 필드 순서 변경 실험
   - 결과: 전환율 +4.1% (유의미)
   - 인사이트: 간단한 것부터 시작하면 완료율 상승

제안: 입력 필드 순서 최적화와 함께 버튼 대비를 높이는 방향을 추천드립니다.
```

### 예시 2: PM의 빠른 데이터 확인

**상황:** PM이 어제 출시한 기능의 사용률을 빠르게 확인하고 싶음

**대화:**
```
사용자: 어제 출시한 '빠른 결제' 기능 사용률이 어때?

재만: '빠른 결제' 기능 현황을 확인했습니다:

📊 출시 후 24시간 데이터:
- 기능 노출 사용자: 12,450명
- 기능 사용 사용자: 1,867명
- 사용률: 15.0%

📈 비교 분석:
- 예상 사용률(기획서): 10%
- 실제 사용률: 15% (목표 대비 +50%)

💡 인사이트:
- 모바일 사용률(18%)이 웹(12%)보다 높음
- 기존 사용자의 사용률이 신규보다 2배 높음

⚠️ 모니터링 포인트:
- 결제 완료율은 기존 대비 동일 (96%)
- 에러 발생률 0.2%로 정상 범위
```

### 예시 3: 비개발 직군의 데이터 접근

**상황:** 마케터가 지난 달 캠페인 효과를 알고 싶음

**대화:**
```
사용자: 지난 달 인스타그램 캠페인 효과가 있었어?

재만: 지난 달 인스타그램 캠페인 분석 결과입니다:

📊 캠페인 성과:
- 캠페인 유입: 8,234명
- 회원가입: 1,423명 (전환율 17.3%)
- 첫 구매: 312명 (구매 전환율 21.9%)

💰 ROI 분석:
- 광고비: 500만원
- 예상 LTV 기반 매출: 1,240만원
- ROAS: 248%

📈 전월 대비:
- 유입 +23%
- 회원가입 전환율 +3.2%p
- 구매 전환율 +1.8%p

💡 인사이트:
효과가 있었습니다! 특히 25-34세 여성 세그먼트에서
전환율이 가장 높았어요 (28.5%).
다음 캠페인에서 이 세그먼트를 더 타겟팅하면 좋을 것 같습니다.
```

## Common Scenarios

### 일일 체크

```
사용자: 오늘 주요 지표 상태 알려줘

재만: 오늘(1/27) 주요 지표 현황입니다:

✅ 정상 범위:
- DAU: 45,230 (전주 대비 +2%)
- 회원가입: 892 (전주 대비 -1%)
- 결제 전환율: 3.2% (전주 동일)

⚠️ 주의 필요:
- 앱 크래시율: 0.8% (전주 0.5% → +60%)
  └ 원인: iOS 17.3 업데이트 영향 추정

모든 핵심 지표는 정상이나, 앱 크래시율 모니터링을 권장드립니다.
```

### 실험 설계

```
사용자: 결제 페이지에서 '쿠폰 자동 적용' 기능을 테스트하고 싶어

재만: 좋습니다! 실험을 설계해 드릴게요.

먼저 몇 가지 질문이 있습니다:
1. 예상하는 효과는 무엇인가요? (예: 결제 완료율 증가)
2. 어느 정도의 개선을 기대하시나요?
3. 실험 기간에 제약이 있나요?

[사용자 답변 후]

📋 실험 계획서:

**가설:**
쿠폰 자동 적용 시 결제 완료율이 5% 이상 증가할 것이다.

**실험 설계:**
- 대조군: 기존 쿠폰 수동 적용
- 실험군: 쿠폰 자동 적용
- 샘플 크기: 각 그룹 5,000명 (95% 신뢰도, 80% 검정력)
- 예상 기간: 7일

**주요 지표:**
- Primary: 결제 완료율
- Secondary: 쿠폰 사용률, 객단가

계획서를 저장할까요?
```

## Tips & Tricks

### 효과적인 질문 방법

**좋은 예:**
- "지난 주 회원가입 전환율이 어떻게 됐어?"
- "결제 퍼널에서 어디서 가장 많이 이탈해?"
- "A/B 테스트 결과를 해석해줘"

**더 좋은 예:**
- "지난 주 회원가입 전환율이 전주 대비 어떻게 변했어?"
- "결제 퍼널에서 모바일 사용자가 어디서 가장 많이 이탈해?"
- "지난 달에 종료된 '버튼 색상' A/B 테스트 결과를 해석해줘"

### 서비스 컨텍스트 활용

서비스 컨텍스트를 잘 설정하면 재만이 더 정확한 답변을 제공합니다:
- 주요 퍼널 정의
- 핵심 지표 및 목표값
- 비즈니스 맥락

## Troubleshooting

### Amplitude 연결 오류

**증상:** "Amplitude 데이터를 조회할 수 없습니다"

**해결:**
1. Amplitude MCP 서버 상태 확인
2. API 키 유효성 확인
3. 프로젝트 ID 확인

### 데이터가 없는 경우

**증상:** "해당 기간에 데이터가 없습니다"

**해결:**
1. 이벤트명이 정확한지 확인
2. 분석 기간 조정
3. 서비스 컨텍스트 재설정 (`[SC]`)

### 느린 응답

**증상:** 분석 응답이 오래 걸림

**해결:**
1. 분석 기간을 좁혀서 재시도
2. 복잡한 쿼리를 단순화
3. 피크 시간 외에 재시도
